# v1.23.2 - Bugfix: update_workflow 400 Bad Request

## 🐛 Bug Fix

### `update_workflow` fails with 400 Bad Request on PUT /workflows/:id

**Root cause:** n8n stores internal properties (`availableInMCP`, `binaryMode`) in workflow settings objects. When `update_workflow` reads the current workflow and echoes the settings back in the PUT payload, the n8n API rejects the request with:

```
400 Bad Request: "request/body/settings must NOT have additional properties"
```

**Fix:** Added `ALLOWED_SETTINGS_KEYS` constant to `client.py` and filter settings at two points:
1. When copying settings from the current workflow into the PUT payload
2. When applying user-provided settings updates

```python
ALLOWED_SETTINGS_KEYS = {
    "executionOrder", "timezone", "callerPolicy", "errorWorkflow",
    "saveDataSuccessExecution", "saveDataErrorExecution",
    "saveManualExecutions", "saveExecutionProgress"
}
```
